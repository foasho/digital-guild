# DIGITAL GUILD SERVICE - サービスコンテキストドキュメント

## 概要

DIGITAL GUILDは、地域経済を活性化させる「冒険者ギルド」を現代に再発明したジョブマッチングプラットフォームです。

自由に働きたい若者（冒険者）と労働者不足に悩む地方（地域の人々）を繋ぎ、Web3技術（JPYC、スマートコントラクト）を活用することで、高速決済と透明性のある信用スコアシステムを実現します。

---

## サービスが解決する2つの構造的課題

### 1. 人と人を隔てる「不信の壁」

| 冒険者の不安 | 地域の人々の不安 |
|-------------|-----------------|
| 給料はすぐもらえる？ | ドタキャンされないか？ |
| ちゃんと評価される？ | ほんとに働けるか？ |
| 未払いリスクはないか？ | 身元は確かか？ |

### 2. 資金が流出し続ける「漏れバケツ」

地域に落ちたお金の最大15%程度（OTA・決済手数料）が、数秒で都市部に吸い出されている現状があります。

**DIGITAL GUILD SERVICEは、価値が循環し続ける「閉じられた生態系」を構築します。**

## コアコンセプト

### 1. ジョブマッチング（掲示板モード / マップモード）

冒険の始まりは『掲示板』から：
1. ジョブ（お仕事）を掲示板から探す
2. 自分に合わせた、AIの『おすすめ』もある
3. 詳細の内容を確認する
4. 応募する
5. ジョブモードで自分のジョブを確認

マップモードでは、あなたがいる『場所』からもジョブを探せます。

### 2. TRUST PASSPORT（信用の証）

冒険者は、信用を示すためにTRUST PASSPORTを使います：

1. ジョブ（お仕事）を完了報告
2. ジョブ実施の実績が自動で記録される
3. 依頼元の評価を受ける
4. 実績や評価はブロックチェーン上に記録
5. TRUST PASSPORTに『信用度』として表示
6. 『信用ポイント』が増えるとランクアップ

**技術基盤**: SBT（Soul Bound Token - 譲渡不能なNFT）とVC（Verifiable Credential - 検証可能なデジタル証明書）

#### 信用スコア計算式
```
TrustScore = クエスト達成数（最大50） + 人間評価平均 × 10
```
- **最大値**: 100
- **クエスト達成数**: 完了したジョブ数（最大50ポイント）
- **人間評価**: 発注者からの星評価（1-5）の平均値 × 10

#### ランク閾値
| ランク | 必要スコア |
|--------|-----------|
| Bronze | 0 - 59 |
| Silver | 60 - 69 |
| Gold | 70 - 79 |
| Platinum | 80 - 100 |

### 3. WALLET（ステーブルコインによる即報酬・即利用）

冒険者は、地域との関わりのためにWALLETを使います：

1. ジョブ（お仕事）の報酬が振り込まれる
2. JPYC対応の地域のお店ですぐに使える
3. JPYC対応の宿泊施設でも使える

**ステーブルコイン（JPYC）を地域通貨のように使えるので、現金化することなくスピーディに利用できます。**

このスピードが「訪れる⇒稼ぐ⇒使う⇒留まる」の体験をスムーズに実現します。

## AIによるインセンティブ設計

地域活性のためにジョブ応募を平準化するインセンティブを発生させる仕組み：

```
AIエージェント ルール例：
- "IF" 天候が"雨"なら → "THEN" 還元率 +5%
- "IF" エリアが "過疎指定地域" → "THEN" 還元率 +10%
- "IF" 曜日が"平日" → "THEN" ランチクーポン
```

冒険者を賢く導くため、プールされた公的補助金からAIエージェントが采配をすることで、投下した予算も1円単位で可視化されます。

## ステークホルダーとの連携

DIGITAL GUILDは競合しません。すべてのプレイヤーをつなぐハブになります：

| パートナー | 役割 |
|-----------|------|
| **JPYC** | ステーブルコイン発行と法的スキーム（技術基盤） |
| **地方銀行** | 法定通貨へのゲートウェイ、KYCによる信用担保、加盟店開拓 |
| **地方自治体** | 公的補助金による介入とその効果のデータ化 |
| **既存事業者** | 商店やホテルなどジョブの依頼元であると同時に冒険者のサポーターにもなる |

### 地方銀行との連携（実現のカギ）

1. **信頼のゲートウェイ**: 銀行口座と直結し、預金保険の対象となる預金トークンのようにJPYCを安心して利用できる金融インフラを構築
2. **地域資金の番人**: この取り組みへの地元企業の参加を促したり、クレジットカード等で流出していた決済手数料収益を地域内に取り戻す
3. **KYCの担い手**: 反社チェック済みの確かな身元保証を提供

## ロードマップ

### Phase 1: 実証実験
- **Scope**: 特定エリア（温泉街など）でのMVP
- **Features**: マッチング、JPYC送金、簡易スコア
- **Goal**: 地域完結モデルの有効性を証明

### Phase 2: 地域実装
- **Scope**: 地銀公式アプリとの機能統合
- **Features**: 加盟店ネットワークの本格拡大
- **Goal**: 1つの県で持続可能なモデルを確立

### Phase 3: 全国展開
- **Scope**: 導入地域間での人材流動（夏は北海道、冬は沖縄）
- **Features**: Trust Passportのポータビリティ化（SBT/VC技術）
- **Goal**: 『信用』をパスポートに日本中を旅して働く世界の実現

## 技術スタック

### Frontend（Phase 1 モック）
- **Framework**: Next.js 16 (App Router)
- **Language**: TypeScript
- **UI**: HeroUI + Tailwind CSS
- **State Management**: Zustand
- **Validation**: Zod
- **Map**: Leaflet
- **AI**: Google Gemini API

### Blockchain（Phase 2以降）
- **Smart Contract**: Solidity (Hardhat)
- **Payment**: JPYC (ERC-20)
- **NFT**: TrustPassport (ERC-721)
