- **Scope**: 導入地域間での人材流動（夏は北海道、冬は沖縄）
- **Features**: Trust Passportのポータビリティ化（SBT/VC技術）
- **Goal**: 『信用』をパスポートに日本中を旅して働く世界の実現

### ステータス: 🔮 将来計画

## 技術スタック ロードマップ

### 現在（Phase 1）

```
Frontend:     Next.js 16 + TypeScript + HeroUI + Tailwind
State:        Zustand
Data:         LocalStorage (Mock)
Map:          Leaflet
AI:           Gemini API
```

### Phase 2

```
Web:          Next.js 16 + TypeScript + HeroUI + Tailwind (Requester UI)
Mobile:       React Native (Expo) + TypeScript (Worker UI)
Backend:      Golang
Database:     PostgreSQL + Prisma / Supabase
Auth:         NextAuth.js / Wagmi
Blockchain:   Solidity + Hardhat + ethers.js
Payment:      JPYC SDK
Realtime:     WebSocket / Supabase Realtime
Infra:        Vercel / AWS
```

### Phase 3

```
Web:          Next.js (Requester UI / Admin)
Mobile:       React Native (Expo) - iOS / Android 両ストア公開
Backend:      Microservices (Kubernetes)
Database:     PostgreSQL + Redis + TimescaleDB
Auth:         DID / VC
Blockchain:   Multi-chain (Ethereum, Polygon, etc.)
AI:           Vertex AI / Custom ML Models
Infra:        Multi-region Cloud
```

## リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| **地銀連携の遅延** | 高 | 複数行と並行交渉、独自決済手段の検討 |
| **法規制の変更** | 高 | 法務チームによる継続的モニタリング |
| **競合サービスの登場** | 中 | 地域密着・信用スコアの差別化 |
| **ブロックチェーン技術の変化** | 中 | マルチチェーン対応、技術柔軟性確保 |
| **ユーザー獲得の遅れ** | 中 | 自治体・地銀との連携によるプロモーション |
